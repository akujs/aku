# Test Quality

Use the findings code TESTQUAL.

Review whether existing tests correctly implement what their descriptions claim, and whether they follow code quality standards. This category focuses on the tests themselves — are assertions correct? Do test names accurately describe what is being tested? Is the test code clear and maintainable? Are there tests that would pass even if the code under test were broken?

This is a companion to the **test-coverage** criteria, which reviews whether the right set of tests exists. This category reviews whether existing tests are correct and well-written; test-coverage reviews whether the right tests exist.

Note: the review unit for this category must include both implementation files and their corresponding test files.
