# Test Coverage

Use the findings code COVERAGE.

Review whether the code under review has an appropriate set of tests. This category focuses on what is tested and what is not — are the right behaviours covered? Are important edge cases exercised? Are there gaps where a regression could go undetected?

This is a companion to the **test-quality** criteria, which reviews whether existing tests are well-written. This category reviews whether the right tests exist; test-quality reviews whether they are correct and maintainable.

Note: the review unit for this category must include both implementation files and their corresponding test files.
